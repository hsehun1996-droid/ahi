# 운영계획변경 한글(HWP) 양식 — 누름틀(필드) 설정 안내

자동 작성이 표의 **정확한 칸**에 들어가도록, 템플릿(`operation_plan_template.hwp`)의
각 칸에 **누름틀(필드)** 을 한 번만 심어두면 됩니다. 셀 병합·머리글 구조와 무관하게
필드 이름으로 정확히 채워지므로, 이전처럼 값이 머리글로 침범하거나 여러 행이 한 칸에
뭉치는 문제가 사라집니다.

> 이 작업은 **한글 PC에서 한 번만** 하면 됩니다. (리눅스/서버 환경에서는 .hwp를 편집할 수 없습니다.)

---

## 1. 누름틀(필드) 넣는 법 (한글에서)

1. 값이 들어갈 **셀 안에 커서**를 둡니다.
2. 메뉴 **입력 → 누름틀** (또는 `입력 → 필드 입력`) 선택.
   - 버전에 따라 단축키 `Ctrl + K` → 누름틀이 있을 수 있습니다.
3. 삽입된 누름틀을 더블클릭(또는 우클릭 → 누름틀 고치기) → **필드 이름**에 아래 표의 이름을 입력.
4. 모든 칸에 대해 반복.

> 팁: 누름틀의 "입력할 내용 안내문(도움말)"에는 아무 글자나 넣어도 됩니다. 자동 작성 시
> 그 자리에 실제 값이 들어가며 안내문은 사라집니다.

---

## 2. 필드 이름 표

### 제목/목적
| 위치 | 필드 이름 |
|------|-----------|
| 단위사업명(제목) | `title` |
| 목적 문구 | `purpose` |

※ `title`/`purpose` 필드를 안 만들면, 기존 예시 문구를 찾아바꾸기로 대체합니다.

### 표1 · 포장개량 세부위치
**계(합계) 행**
| 칸 | 필드 이름 |
|----|-----------|
| 개소 수 | `t1_total_count` |
| 연장 합계 | `t1_total_len` |
| 당초 합계 | `t1_total_init` |
| 변경 합계 | `t1_total_change` |
| 증감 합계 | `t1_total_delta` |

**데이터 행** — `i` 는 행 번호(1, 2, 3 …). 만들어 둔 행 수만큼 자동 인식됩니다.
| 칸 | 필드 이름 |
|----|-----------|
| 노선명 | `t1_route_i` |
| 위치 | `t1_loc_i` |
| 차로 | `t1_lane_i` |
| 연장 | `t1_len_i` |
| 당초 | `t1_init_i` |
| 변경 | `t1_change_i` |
| 증감 | `t1_delta_i` |
| DI등급 | `t1_di_i` |
| 비고 | `t1_note_i` |

예: 1행은 `t1_route_1`, `t1_loc_1` … / 2행은 `t1_route_2`, `t1_loc_2` … 식으로.

### 표2 · 소요예산 산출내역
`i` = 행 번호(1, 2, …)
| 칸 | 필드 이름 |
|----|-----------|
| 사업내용 | `t2_content_i` |
| 사업량(km) | `t2_qty_i` |
| 단가 | `t2_price_i` |
| 공사비 | `t2_cost_i` |
| 비고 | `t2_note_i` |

### 표3 · 예산변경(안)
| 칸 | 필드 이름 |
|----|-----------|
| 단위사업명 | `t3_name` |
| 당초 | `t3_init` |
| 변경 | `t3_change` |
| 증감 | `t3_delta` |
| 비고 | `t3_note` |

---

## 3. 행 개수 규칙 (중요)

- 프로그램은 **템플릿에 만들어 둔 필드 개수만큼만** 행을 채웁니다.
  - 데이터가 적으면 → 남는 행 필드는 **빈칸**으로 비워집니다.
  - 데이터가 많으면 → 필드 개수까지만 채우고, **초과분은 누락**됩니다(로그에 경고).
- 따라서 표1·표2는 **예상되는 최대 행 수만큼** 미리 행을 만들고, 각 칸에 `_1, _2, …`
  번호 필드를 넣어두는 것을 권장합니다.
- 행 번호는 **1부터 끊김 없이 연속**이어야 합니다. (예: `t1_route_1`, `t1_route_2`,
  `t1_route_3` … — 중간에 비면 거기서 멈춥니다.)

---

## 4. 동작 방식 요약

- 템플릿에 위 필드가 **하나라도** 있으면 → **필드 방식**으로 정확히 채웁니다(권장).
- 필드가 **전혀 없으면** → 과거 호환을 위해 옛 셀-이동 방식으로 동작합니다
  (병합 양식에서는 정렬이 깨질 수 있으므로 권장하지 않음).
- 최초 1회는 한글 설치 PC에서 결과를 꼭 확인하세요.
